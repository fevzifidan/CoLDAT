# MobileSAM ONNX Models

Place the MobileSAM encoder and decoder ONNX model files here:

- `mobile_sam_encoder.onnx` — Image encoder model (~6-8 MB)
- `mobile_sam_decoder.onnx` — Prompt decoder model (~4-6 MB)

## Model Sources

These models are generated by exporting the MobileSAM PyTorch model to ONNX format.
The recommended source is the official MobileSAM repository:
https://github.com/ChaoningZhang/MobileSAM

## Export Instructions

```bash
# Install MobileSAM
git clone https://github.com/ChaoningZhang/MobileSAM
cd MobileSAM

# Export encoder
# Export encoder
.venv/scripts/python scripts/export_onnx_model.py --model-type vit_t --checkpoint ./weights/mobile_sam.pt \ --output ./export/encoder.onnx --encoder-only

# Export decoder
.venv/scripts/python scripts/export_onnx_model.py --model-type vit_t --checkpoint ./weights/mobile_sam.pt \ --output ./export/decoder.onnx --decoder-only

# Copy the exported files to this directory
cp ./export/encoder.onnx /path/to/public/models/mobile_sam_encoder.onnx
cp ./export/decoder.onnx /path/to/public/models/mobile_sam_decoder.onnx
```

## Model Architecture

- **Encoder:** ViT-Tiny (Vision Transformer) based image encoder
  - Input: 1024×1024×3 (padded, normalized)
  - Output: 256×64×64 embedding tensor

- **Decoder:** Mask decoder with prompt encoding
  - Inputs: image embedding, point prompts (coordinates + labels)
  - Output: 1024×1024 binary mask logits

## Notes

- Both models must be in ONNX opset 17 or later
- The worker expects the models to be served as static assets by Vite
- File paths are configured in `src/features/annotation/tools/sam/useSamOrchestrator.ts`
